Syntax
public class CommandGroup : Command, INamedSendable, ISendable, ITableListener
Remarks

Commands in a CommandGroup are added using the AddSequential(Command) method and are called sequentially. CommandGroup are themselves Command and can be given to other CommandGroup

CommandGroup will carry all of the requirements of their Command Additional requirements can be specified by calling Requires(Subsystem) normally in the constructor.

Command Groups can also execute commands in parallel, simply by adding them using AddParallel(Command).

AddParallel(Command)

Adds a new Command to the group.

Declaration
public void AddParallel(Command command)
Parameters
Type Name Description
Command command

The Command to be added.
Remarks

The Command will be started after all the previously added Command.

Instead of waiting for the child to finish, a CommandGroup will have it run at the same time as the subsequent Command. The child wil run either until it finishes, a new child with conflicting requirements is started, or the main sequence runs a Command with conflicting requirements. In the latter two cases, the child will be canceled even if it says it can't be interrupted.

Note that any requirements thee given Command has will be added to the group. For this reason, a Command requirements cannot be changed after being added to a group.

It is recommended that this method be called in the constructor.
