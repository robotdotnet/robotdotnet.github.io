Class Scheduler

The Scheduler is a singleton which holds the top-level running m_commands.

Syntax
public class Scheduler : INamedSendable, ISendable
Remarks

It is in charge of both calling the command's WPILib.Commands.Command.Run method and to make sure that there are no two command with conflicting requirements running.

If is fine if teams with to take control of the Scheduler themselves. All that needs to be done is to call Instance often to have Command function correctly. However, this is already done for you if you use the CommandBased Robot Template.
