Syntax
public class Counter : SensorBase, IDisposable, ICounterBase, IPIDSource, ILiveWindowSendable, ISendable
Remarks

This is a general purpose class for counting repetitive events. It can return the number of counts, the period of the most recent cycle, and detect when the signal being counted has stopped by supplying a maximum cycle time.

All counters will immediately start counting - Reset() them if you need them to be zeroes before use.

UpdateWhenEmpty

Sets whether you want to continue updating the event timer output when there are no samples captured.

Declaration
public bool UpdateWhenEmpty { set; }
Property Value
Type Description
Boolean
Remarks

The output of the event timer has a buffer of periods that are averaged and posted to a register on the FPGA. When the timer detects that the event source has stopped (based on the MaxPeriod) the buffer of samples to be averaged is emptied. If you enable the update when empty, you will be notfied of the stopped source and the event time will report 0 samples. If you disable update when empty, the most recent average will remain on the output until a new sample is acquired. You will never see 0 samples output (except when there have been no events since an FPGA reset) and you will likely not se the stopped bit become true (since it is updated at the end of an average and there are no samples to average.)
