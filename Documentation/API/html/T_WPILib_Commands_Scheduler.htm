Remarks
It is in charge of both calling the command's Run() method and to make sure that there are no two command with conflicting requirements running.

If is fine if teams with to take control of the Scheduler themselves. All that needs to be done is to call Scheduler.Instance. often to have Commands function correctly. However, this is already done for you if you use the CommandBased Robot Template.
