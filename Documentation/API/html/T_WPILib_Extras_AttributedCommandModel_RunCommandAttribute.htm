Remarks
To set up required subsystems, commands can have a constructor that takes subclasses of Subsystem that it requires. Note: this must not have any non-Subsystem-subclass parameters.
If there does not exist a constructor that takes subsystems, the registration will call the default constructor, which must exist in this situtation.
