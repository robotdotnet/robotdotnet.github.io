Remarks
The Command will be started after all the previously added commands.

Instead of waiting for the child to finish, a CommandGroup will have it run at the same time as the subsequent Commands. The child wil run either until it finishes, a new child with conflicting requirements is started, or the main sequence runs a Command with conflicting requirements. In the latter two cases, the child will be canceled even if it says it can't be interrupted.

Note that any requirements thee given Command has will be added to the group. For this reason, a Command's requirements cannot be changed after being added to a group.

It is recommended that this method be called in the constructor.
