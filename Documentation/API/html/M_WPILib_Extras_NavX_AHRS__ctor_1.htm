Remarks
This constructor should be used if communicating via either TTL UART or USB Serial Interface.

Note that the serial interfaces can communicate either processed data, or raw data, but not both simultaneously. If simultaneous processed and raw data are needed, use one of the register-based interfaces (SPI or I2C). The default is processed data.

The update rate may be between 4 Hz and 60 Hz, representing the number of updates per second sent by the sensor. Note that increasing the update rate may increase the CPU utilization.
